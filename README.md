# Principios de Diseño de Software: Ejemplo (C++)

El ejemplo muestra tres principios fundamentales en el diseño de software:

- Cohesión (Cohesion)
- Acoplamiento (Coupling)
- Separación de responsabilidades (Separation of Concerns)

El ejemplo comienza con un sistema mal diseñado y lo refactoriza progresivamente para mejorar el diseño.

## Estructura del repositorio

### 00_bad_design
Implementación inicial con múltiples problemas de diseño.

### 01_cohesion
Refactorización enfocada en mejorar la **cohesión**, asignando responsabilidades claras a cada clase.

### 02_coupling
Ejemplo de reducción del **acoplamiento**, minimizando dependencias entre componentes.

### 03_separation_of_concerns
Organización del sistema siguiendo el principio de **separación de responsabilidades** mediante capas arquitectónicas.

### 04_final_design

Arquitectura limpia, escalable y mantenible

## Ruta de aprendizaje

1. Comenzar con `00_bad_design`
2. Observar los problemas en el diseño
3. Aplicar mejoras de cohesión
4. Reducir el acoplamiento
5. Introducir la separación de responsabilidades
6. Lograr la arquitectura final
